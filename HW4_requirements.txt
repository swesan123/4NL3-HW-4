## Step-by-step checklist (assigned task)
### 1) Dataset familiarization
- Get the dataset for your assigned group (Avenue: “Group Assignments A4.txt”).
- Understand label space, class balance, text fields, train/test splits (if any).

### 2) Fine-tune TWO pretrained transformer models (§3.1)
- Pick 2 different pretrained transformer models (examples: BERT, RoBERTa, DistilBERT, DeBERTa, T5, Electra, ModernBERT, etc.).
- Constraint: models must NOT already be fine-tuned on *your exact task/dataset*.
  - You may use models fine-tuned on other tasks, but verify there’s no overlap with your task.
- Hardware note: avoid multi-billion parameter models on free Colab; “BERT-base sized” is typical; distilled/small/tiny variants are ok.
- Data split rule:
  - If dataset has no predefined split (or everything is in train), create your own:
    - 80% train / 20% test
    - (optional) 80/10/10 train/val/test if tuning hyperparams
- Train on train split, predict on test split.
- Report “appropriate evaluation metrics” for your task.
  - Note: you can compute metrics locally for train/val, but test-set evaluation may rely on the Colab leaderboard (per PDF).

### 3) Zero-shot classification with language models (§3.2)
- Use prompt engineering to get a pretrained generative LM to output class labels (no fine-tuning).
- Use training examples to craft prompts; try multiple prompts; pick the best prompt and use ONLY that prompt on the test set.
- Option (local model): instead of generating text, you may compare next-token probabilities:
  - Choose label via higher P(label_token | prompt), e.g., P(yes|prompt) vs P(no|prompt)
- Use TWO different models for zero-shot.
  - HuggingFace text-generation models are allowed (watch memory requirements).
  - Allowed alternatives: optimized local runners like Llama.cpp; or Cloud API LLMs (GPT-4 / Gemini), but not free.
  - DO NOT do manual chat copy/paste per test example; use code + APIs.
  - Another mentioned option: NLI-based zero-shot classification (as per referenced method in PDF).
- Evaluate on test data using the SAME metrics used for fine-tuning so results are comparable.

### 4) Baselines (§3.3)
- Train a bag-of-words (BoW) classifier using scikit-learn (or similar).
  - Allowed features: TF-IDF, word vectors, other non-neural features.
  - Not allowed: neural models with recurrence or attention.
  - Allowed model families: logistic regression, naive bayes, SVM, etc.
- Also compute simple sanity baselines:
  - Majority-class baseline (predict most frequent label)
  - Random baseline (random label)
  - If leaderboard uses single-class F1: consider “target-class baseline” (always predict target/positive class)
- Compare your models against these baselines; choose the most competitive/simple baselines for your task context.
